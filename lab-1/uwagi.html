Moje uwagi dotyczące formularzy w HTML

Formy w HTML to standardowy sposób do zbierania danych użytkowników i przesyłania ich do serwera

Formę HTML tworzy się za pomocą elementa <form>, i zazwyczaj zawiera kontrolki input takie jak:

  • <input> (text, password, email, checkbox, radio, file, etc.)
  • <textarea>
  • <select>
  • <button>

Na przykład:

        
          <form action="/login" method="post">
            <input type="email" name="email" required>
            <input type="password" name="password" required>
            <button type="submit">Login</button>
          </form>

W czasie gdy użytkownik naciska na klawisz "Submit" konwertuje dane w JSON i browser przesyła dane do serwera przez HTTP.

Kiedy należy używać HTML formularzy?

  1. Zbieranie danych użytkowników

    2. Formularze są najlepszym wyborem kiedy zapada potrzeba zbierania danych użytkowników:

    • Logowanie / Rejestracja
    • Formularze kontaktowe
    • Pola wyszukiwania
    • Strony płatności
    • Ustawienia konta
    • Komunikacja zwrotna / sondaże

  2. Wysyłanie danych do serwera

    3. Formy są idealne do:

    • Przesyłania danych do serwera za pomocą GET i POST
    • Natywnego zachowywania przesyłania
    • Kompatybilności bez JavaScript`u

    Przykład:

              
            <form method="get" action="/search">
              <input name="q">
            </form>

  3. Kiedy dostępność (ang. accessibility) ma znaczenie

    4. HTML formularze:

    • Mają do czynienia z czytnikami ekranowymi (ang. screen readers)
    • Wspierają nawigację przez klawiaturę
    • Pozwalają na labels (<label for="">)
    • Zapewniają natywną informację zwrotną o błędach
  4. Wbudowana walidacja

    5. HTML5 oferuje wbudowaną walidacje:

              
            <input type="email" required minlength="5">

    Korzyści

    • Nie ma potrzeby w JavaScripcie
    • Spójne doświadczenie użytkowania (ang. User Experience - UX)
    • Mniej kodu

Kiedy nie należy używać HTML formularzy?

  1. Kiedy nie ma potrzeby wprowadzenia danych od użytkowika

    Nie należy używać formularzy do:

    • Menu nawigacyjnych
    • Layout`u
    • Przecisków, które nie przesyłają danych

  2. Podczas wykonywania czynności wyłącznie dla klienta

    Jeżeli czynność:

    • Nie przesyład danych
    • Przełacza tylko stan UI
    • Otwiera okna modalne
    • Wywoła animacje

    Do tego trzeba wykorzystać przyciski i JavaScript

  3. W przypdaku korzystania z nowoczesnych API z pełną kontorlą JS

    Jeżeli aplikacja:

    • Używa Fetch API lub Axios do przesyłania danych
    • Przesyła tylko JSON
    • Posiada pełną logikę SPA (React, Angular, Vue)

    W takich przypdkach można uniknąć przesyłania formularzy natywnych, lecz i nadal zachować <form> ze względów semantycznych

    Typowy wzór SPA:

    Zapobiega domyślnemu przesyłaniu, lecz zachowuje dostępność

  4. W przypdaku niewłaściwego użycia formularzy do stylizacji

    Formularze nigdy nie powinny być używane:

    • Jako kontenery do layout`u CSS
    • Do grupowania elementów wizualnie

    Zamiast tego należy używać

    • <div>
    • <section>
    • <fieldset> (tylko formularzy wewnętrzne)
